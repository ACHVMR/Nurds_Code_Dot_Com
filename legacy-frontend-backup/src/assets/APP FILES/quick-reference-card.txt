## CODE DEPLOYMENT (Every deployment)

**Follow these steps EXACTLY:**

```bash
# 1. Feature branch
git checkout -b feature/my-feature

# 2. Make changes + test locally
pytest tests/ -v

# 3. Build & push
docker build -t achievemore/ii-agent:my-feature .
docker tag achievemore/ii-agent:my-feature \
    gcr.io/$GCP_PROJECT_ID/ii-agent:my-feature
docker push gcr.io/$GCP_PROJECT_ID/ii-agent:my-feature

# 4. Deploy to staging
gcloud run deploy ii-agent-staging \
    --image gcr.io/$GCP_PROJECT_ID/ii-agent:my-feature
sleep 10

# 5. Test staging
curl -X POST https://ii-agent-staging-xxx.run.app/test/resume-optimizer \
    -H "Authorization: Bearer ${STAGING_API_KEY}" \
    -d '{"resume_url":"test.pdf"}'

# 6. Deploy to production (via CLI - safest)
achievemore deploy production

# 7. Verify
achievemore status | grep ii-agent

# 8. Commit code
git add -A && git commit -m "feature: [description]"
git push origin feature/my-feature
# Create PR on GitHub
```

**If anything fails at any step:**
1. STOP (don't continue)
2. Rollback to previous version (if in production)
3. Debug in staging
4. Try again

## COST TRACKING (Weekly, Friday 5pm)

```bash
# Run weekly cost analysis
achievemore logs all 1000 | grep "cost" | \
    awk '{sum += $NF} END {print "Total cost: $" sum}'

# Check by model
sudo -u postgres psql -d achievemore_db << 'EOF'
SELECT model, ROUND(SUM(cost)::numeric, 2) as weekly_cost
FROM model_invocations
WHERE created_at > NOW() - INTERVAL '7 days'
GROUP BY model
ORDER BY weekly_cost DESC;
EOF

# Monthly trend
# If cost rising:
# 1. Check which models are expensive
# 2. Update routing in Terminal Manual Â§ 5
# 3. Increase cache TTL (cheaper)
# 4. Reduce token limits (faster + cheaper)
```

## PERFORMANCE ISSUES (If p99 latency > 5 seconds)

**Quick diagnostics:**

```bash

free -h          # Memory
df -h            # Disk
top              # CPU (q to quit)

# 2. Check database
sudo -u postgres psql -d achievemore_db << 'EOF'
SELECT query, calls, mean_exec_time FROM pg_stat_statements 
ORDER BY mean_exec_time DESC LIMIT 5;
EOF

# 3. Check model providers
# Visit: 
# - https://status.openai.com (OpenAI)
# - https://www.google.com/appsstatus (Google)
# - https://status.anthropic.com (Anthropic)

# 4. Profile a request
curl -v http://localhost:8000/api/resume/optimize \
    -H "X-Debug: true" | tee request.log

# 5. Fix based on findings
# Database slow? Add index (Platform DevOps Â§ 10)
# Memory high? Reduce cache size
# Model slow? Switch provider
```

## THE GOLDEN RULES

1. **Always follow Terminal Manual Â§ 1.2 when deploying**
   - Don't skip steps
   - Don't merge to main without this

2. **Always escalate if unsure**
   - Better to wake up senior eng than cause outage
   - It's literally their job to help

3. **Always document everything**
   - Incident report (even if resolved quick)
   - Update to Terminal Manual (if new procedure)
   - Slack post (so team knows what happened)

4. **Always test in staging first**
   - Never deploy directly to production
   - Always verify staging works

5. **Always check Terminal Manual Â§ 2 first**
   - Before doing anything else
   - 95% of incidents are in there

6. **Always optimize for user impact**
   - Smallest fix to get users back
   - Debug deeper afterward
   - Temporary fix is better than no fix

7. **Always be ready to rollback**
   - Keep previous version in mind
   - Know how to deploy previous
   - Do it fast if needed
